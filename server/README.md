# Server API

Кратко: это Fastify-бэкенд для объявлений с AI-эндпоинтами.
- Работа с объявлениями: список, карточка, обновление.
- AI-эндпоинты:
  - улучшение описания;
  - рекомендация цены с учетом веб-сигналов рынка.
- AI-ответы унифицированы: всегда `{ "message": "..." }`.

## Требования

- Node.js 20+
- npm
- Запущенный Ollama (локально)
- Загруженная модель, указанная в `.env` (по умолчанию `llama3`)

## Быстрый запуск

1. Установить зависимости:

```bash
npm install
```

2. Убедиться, что Ollama запущен и модель загружена:

```bash
ollama pull llama3
ollama serve
```

3. Проверить `.env` (пример):

```env
OLLAMA_URL=http://127.0.0.1:11434/api/generate
OLLAMA_MODEL=llama3
OLLAMA_TIMEOUT_MS=30_000
OLLAMA_TEMPERATURE=0.1
```

4. Запустить сервер:

```bash
npm start
```

По умолчанию сервер стартует на `http://localhost:8080`.

## Основные эндпоинты

- `GET /items` — список объявлений
- `GET /items/:id` — карточка объявления
- `PUT /items/:id` — обновление объявления
- `POST /ai/improve-description` — AI-улучшение описания
- `POST /ai/suggest-price` — AI-рекомендация цены

## Формат AI-ответов

Оба AI-эндпоинта возвращают:

```json
{
  "message": "..."
}
```

## Полезные env-переменные

- `PORT` — порт API (по умолчанию `8080`)
- `WEB_SEARCH_ENABLED` — включение веб-поиска для цен (по умолчанию включен)
- `WEB_SEARCH_TIMEOUT_MS` — timeout запросов веб-поиска
- `WEB_SEARCH_RESULTS_LIMIT` — лимит результатов поиска
- `WEB_SEARCH_PAGE_FETCH_LIMIT` — сколько страниц дополнительно парсить
